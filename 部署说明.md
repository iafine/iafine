# 部署到你的 GitHub 主页(4 步)

## 0. 先替换变量

在 `README.md` 里把所有 `YOUR_GITHUB_USERNAME` 全局替换成你的 GitHub 用户名。
顺手可改:`YOUR_LINKEDIN`、`YOUR_BLOG_URL`、banner 里的 `text=Yang%20Li`、打字机标语、技术栈图标列表。

## 1. 建同名仓库

在 GitHub 新建一个 **public** 仓库,仓库名 **必须等于你的用户名**(例如用户名是 `yangli` 就建 `yangli/yangli`)。同名才会绑定到个人主页。

## 2. 推送文件

把本目录的 `README.md` 和 `.github/` 一起放进仓库并 push:

```bash
git init
git add README.md .github
git commit -m "cool profile"
git branch -M main
git remote add origin https://github.com/YOUR_GITHUB_USERNAME/YOUR_GITHUB_USERNAME.git
git push -u origin main
```

## 3. 开 Actions 写权限(贪吃蛇必需)

仓库 **Settings → Actions → General → Workflow permissions → 选 "Read and write permissions" → Save**。
不设这一步,贪吃蛇 Action 推 `output` 分支会 403。

## 4. 手动跑一次贪吃蛇

仓库 **Actions 标签页 → 左侧 Generate Snake → Run workflow**,等绿勾。
之后每 12 小时自动刷新。

---

## 验证

打开 `github.com/你的用户名`:
- banner、打字机、技术栈徽章、summary-cards 统计卡(5 张)、活动折线图 → 实时服务,立即显示。
- 贪吃蛇 → 等第 4 步 Action 跑完生成 `output` 分支后才显示。

## 常见问题

- **贪吃蛇红叉** → 90% 是第 3 步权限没开。
- **某张卡裂图/不显示** → 第三方服务限流,刷新或稍后即可;summary-cards / 活动图首次访问会冷启动,等几秒重刷。
- **换配色** → summary-cards 的 `theme=tokyonight` 可换 `github_dark` / `default` 等;活动图的 `theme=react-dark` 可换 `github-compact` / `dracula` 等。
- **想更简洁** → 直接删 README 里不想要的模块即可,互不影响。
